## 5-Minute Deployment

### 1. Install Prerequisites

```bash
# Azure CLI
brew install azure-cli  # macOS
# or download from https://aka.ms/azure-cli

# JSON processor
brew install jq

# Verify
az --version
jq --version
```

### 2. Authenticate

```bash
az login
az account show  # Verify
```

### 3. Deploy

```bash
cd cgr-mvp

# Make scripts executable
chmod +x scripts/deploy.sh scripts/validate-deployment.sh

# Deploy to dev (takes ~5 minutes)
./scripts/deploy.sh dev eastus
```

### 4. Validate

```bash
./scripts/validate-deployment.sh dev
```

### 5. View Output

Script will display:

```
App Service URL:    https://cgr-dev-api.azurewebsites.net
Health Check:       https://cgr-dev-api.azurewebsites.net/api/health
CosmosDB Account:   cgr-dev-db
Database:           cgr_platform
```

## What Gets Created

### Azure Resources

| Resource | Name | Details |
|----------|------|---------|
| **Resource Group** | `cgr-{env}-rg` | Container for all resources |
| **CosmosDB Account** | `cgr{env}db` | NoSQL database |
| **CosmosDB Database** | `cgr_platform` | Main database |
| **App Service Plan** | `cgr-{env}-asp` | Compute resources |
| **App Service** | `cgr-{env}-api` | Hosted Node.js application |

### CosmosDB Collections

```
posts       (partition: /id)       — Published posts
comments    (partition: /id)       — Comments and replies
reactions   (partition: /id)       — Emoji reactions
users       (partition: /id)       — User profiles
approvals   (partition: /id)       — Approval workflow
audit_logs  (partition: /timestamp) — Immutable audit trail
analytics   (partition: /date)     — Daily engagement metrics
```

### Health Check Endpoints

```
GET /api/health        → Basic health status
GET /api/health/live   → Liveness probe (is process running?)
GET /api/health/ready  → Readiness probe (is app ready for requests?)
GET /api/health/detailed → Full diagnostics (Node.js, memory, CosmosDB, env vars)
```

## After Deployment

### 1. Add Health Module to API

The health check module is ready. Add to your main API module:

```typescript
// apps/api/src/app.module.ts
import { HealthModule } from './health/health.module';

@Module({
  imports: [
    // ... other imports
    HealthModule,
  ],
})
export class AppModule {}
```

### 2. Build & Deploy Application

```bash
# Build
npm run build

# Test locally
npm start

# Deploy to App Service (see DEPLOYMENT.md for options)
az webapp deployment source config-zip \
  --resource-group cgr-dev-rg \
  --name cgr-dev-api \
  --src dist.zip
```

### 3. Test Health Endpoints

```bash
APP_URL=$(az webapp show \
  --resource-group cgr-dev-rg \
  --name cgr-dev-api \
  --query defaultHostName \
  --output tsv)

curl https://$APP_URL/api/health
curl https://$APP_URL/api/health/detailed
```

## Deployment for Each Environment

### Development

```bash
./scripts/deploy.sh dev eastus
./scripts/validate-deployment.sh dev
```

- SKU: B1 (basic)
- CosmosDB: 400 RU/s
- Cost: ~$40/month
- Mock auth enabled

### Staging

```bash
./scripts/deploy.sh staging westeurope
./scripts/validate-deployment.sh staging
```

- SKU: S1 (standard)
- CosmosDB: 600 RU/s
- Cost: ~$100/month
- Production-like settings

### Production

```bash
./scripts/deploy.sh prod westeurope
./scripts/validate-deployment.sh prod
```

- SKU: P1V2 (premium)
- CosmosDB: 1000 RU/s
- Cost: ~$150-200/month
- Full monitoring and auto-scale ready

## Troubleshooting

### "Azure CLI not found"
→ Install: `brew install azure-cli`

### "Not authenticated with Azure"
→ Run: `az login`

### "Health endpoint not responding"
→ Check logs: `az webapp log tail --resource-group cgr-dev-rg --name cgr-dev-api`

### "CosmosDB connection failed"
→ Verify credentials: `az cosmosdb keys list --resource-group cgr-dev-rg --name cgr-dev-db`
