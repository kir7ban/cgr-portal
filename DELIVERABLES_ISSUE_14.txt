================================================================================
                             KEY FEATURES
================================================================================

FEATURE: Edit Published Posts
  - Only post creators can edit their own posts
  - Edits automatically trigger re-approval workflow
  - Post state changes: PUBLISHED → SUBMITTED → PUBLISHED
  - Validates all content and media before applying changes
  - Creates new submission in approval queue
  - Preserves original proposed audience

FEATURE: Revision History Tracking
  - Sequential revision numbering (1, 2, 3...)
  - Captures complete before/after state
  - Automatic change summary generation
  - Custom change summaries supported
  - Links revision to approval submission
  - Timestamps and user tracking
  - Historical diffs available for compliance

FEATURE: Audit Trail Integration
  - Two immutable audit entries per edit
  - Edit action logging: "edit_published_post"
  - Revision creation logging: "create_revision"
  - Full timestamps and actor identification
  - Complies with 3-year retention requirement
  - Admin-only read access
  - Searchable and filterable

FEATURE: Content Validation
  - Title: Non-empty string
  - Content: Non-empty string
  - Images: Max 3, types (jpeg/png/gif/webp), max 5MB each
  - Video: Linked only (youtube/internal), no direct uploads
  - Documents: Supported types (pdf/docx/xlsx/xls/doc), max 10MB each

FEATURE: Access Control
  - Comms Officers: Can edit own published posts
  - Admins: Cannot edit (approval role only)
  - Employees: Read-only access
  - ForbiddenException for unauthorized users
  - Clear error messages

                         PERFORMANCE METRICS
================================================================================

Operation Complexity:
  - editPublishedPost(): O(1) for in-memory operations
  - getRevisionHistory(): O(r) where r = number of revisions
  - getRevision(): O(r) linear search
  - getRevisionCount(): O(1) map lookup

Expected Performance:
  - Edit operation: <200ms
  - Audit logging: <50ms
  - Submission creation: <100ms
  - Total edit flow: <350ms

Scalability:
  - Per-post revision history: O(r * e) where r=revisions, e=edit size
  - Audit entries: 2 per edit (fixed size)
  - Recommended indexes: /postId, /editedAt, /revisionNumber, /editedBy
