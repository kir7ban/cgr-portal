OVERVIEW
================================================================================

Enables users to:
- View cached posts when offline
- Create/edit posts while offline (queued for sync)
- Automatically sync changes when connection restored

================================================================================
KEY FEATURES
================================================================================

1. AUTOMATIC POST CACHING
   - 50 posts per department limit
   - Least Recently Used (LRU) eviction
   - Indexed by state, department, creation date
   - Automatic cleanup

2. FEED RESPONSE CACHING
   - 1-hour TTL (configurable)
   - Auto-expiration checking
   - Cache key: userId + audiences + pagination
   - Offline feed viewing

3. OFFLINE SYNC
   - Queue posts: create, update, delete, submit
   - Maximum 3 retries per post
   - Automatic trigger on reconnect
   - Manual sync trigger available
   - Comprehensive error handling

4. NETWORK STRATEGIES
   - API endpoints: Network-first (try network, fallback cache)
   - Static assets: Cache-first (check cache, fallback network)
   - HTML pages: Stale-while-revalidate (cached now, fresh later)
   - Custom fallback responses

5. STATUS MONITORING
   - Online/offline detection
   - Sync status callbacks (pending, syncing, completed, error)
   - Pending changes counter
   - Cache statistics (total posts, pending, feeds)

6. BROWSER COMPATIBILITY
   Chrome 40+  - Full support (incl. Background Sync)
   Firefox 44+ - Full support (except Background Sync)
   Safari 11.1+ - Full support (except Background Sync)
   Edge 17+    - Full support (incl. Background Sync)
   IE          - Not supported

================================================================================
TECHNICAL SPECIFICATIONS
================================================================================

IndexedDB SCHEMA:
  posts
    - id (keyPath)
    - state (indexed)
    - dept (indexed)
    - createdAt (indexed)
    - cachedAt (indexed)
    - 50 posts/dept max (LRU eviction)

  pendingPosts
    - id (keyPath)
    - action: 'create' | 'update' | 'delete' | 'submit'
    - data: serialized post data
    - timestamp (indexed)
    - retryCount (indexed, max 3)

  feeds
    - cacheKey (keyPath)
    - userId (indexed)
    - items: post array
    - expiresAt: 1 hour TTL

CACHE API (Service Worker):
  - STATIC_CACHE: Static assets (index.html, favicon, manifest)
  - DYNAMIC_CACHE: Dynamic assets (JS, CSS, images)
  - API_CACHE: API responses (/api/*)

NETWORK POLICIES:
  /api/* → Network-first
  Static assets → Cache-first
  HTML pages → Stale-while-revalidate
  Unknown → Network-first

CONFIGURATION (Adjustable):
  - CACHE_DURATION_MS: 1 hour (IndexedDB)
  - MAX_POSTS_PER_DEPT: 50 (IndexedDB)
  - MAX_RETRIES: 3 (Offline Sync)
  - RETRY_DELAY_MS: 2000ms (Offline Sync)

================================================================================
USAGE EXAMPLES
================================================================================

1. INITIALIZE IN APP ROOT

   import { CacheService } from '@/services/cache';

   useEffect(() => {
     CacheService.initialize();
   }, []);

2. CACHE POSTS AFTER LOADING

   const response = await api.getFeed(page, pageSize);
   await CacheService.cachePosts(response.items, userDept);

3. FALLBACK TO CACHE WHEN OFFLINE

   try {
     return await api.getFeed(page, pageSize);
   } catch (error) {
     if (CacheService.isOffline()) {
       const cached = await CacheService.getFeed(cacheKey);
       if (cached) return cached;
     }
     throw error;
   }

4. QUEUE OFFLINE CHANGES

   if (CacheService.isOffline()) {
     await CacheService.queuePostSync(postId, 'create', {
       title, content
     });
   }

5. USE REACT HOOK FOR STATUS

   const { isOffline, pendingCount, sync } = useOfflineCache();
